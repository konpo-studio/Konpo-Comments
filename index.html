Notes

Figma-style comments on any WIP site.

One script tag. No login. Click any element, leave a note, resolve it — then hand it straight to Claude Code.

1 Add your name 2 Click an element 3 Comment 4 Resolve

Drop this into any project

<script src="https://kp-comments.vercel.app/embed.js" defer></script>

That's it — one line. Works in plain HTML, Next.js (drop it in your layout), and Figma-Make exports. Comments are namespaced by the page's domain automatically; add data-project="name" to share one thread across URLs.

What's inside

Pin to any element

Click anything on the page and the pin sticks to it — numbered, just like Figma. Hit C to start, Enter to post.

Comments panel

Every note in one side panel, grouped by page and filterable by open vs. resolved. Click one to jump to it — even across routes.

Copy for Claude

Export any thread as a precise prompt — element, selector, route, and the ask — so Claude Code finds the spot and fixes it fast.

Resolve & persist

Resolve with a little purple confetti. Threads save to a shared backend, so everyone on the link sees the same comments.

Keyboard-first

K show / hide the bar · C comment · ←/→ jump between comments · Enter post · Esc cancel. The bar is hidden by default so pins stay out of the way while you iterate.

Just add a name

No accounts, no setup. Type a name, get an avatar, start commenting. A light, frosted bar that stays out of your way.

Use it in Webflow — staging only

<script
  src="https://kp-comments.vercel.app/embed.js"
  data-project="my-webflow-site"
  data-staging
  defer></script>

Paste into Site Settings → Custom Code → Footer, then Publish. Webflow ships the same custom code to your *.webflow.io staging URL and your live domain — so data-staging lets Notes show up only on staging and quietly disappear on production, automatically, every publish. Got a custom staging domain? Swap in data-allow-hosts="staging.yoursite.com".

Site-wide custom code needs a paid Webflow Site plan — on free, paste the same tag into an Embed element on a page. Custom code doesn't run inside the Designer canvas, so test on the published .webflow.io link.
